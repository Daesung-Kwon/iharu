# Expo Go vs 실제 빌드 - 알림 기능 비교

## 🔍 현재 앱의 알림 사용 현황

코드를 확인한 결과, 이 앱은 **로컬 알림(Local Notifications)**만 사용합니다.

### 사용 중인 기능
- ✅ **로컬 알림 스케줄링** (`scheduleNotificationAsync`)
- ✅ **로컬 알림 취소** (`cancelScheduledNotificationAsync`)
- ✅ **알림 권한 요청** (`requestPermissionsAsync`)
- ✅ **Android 알림 채널 설정**

### 사용하지 않는 기능
- ❌ **원격 푸시 알림** (Remote Push Notifications)
- ❌ **FCM/APNs 토큰**
- ❌ **서버에서 보내는 푸시 알림**

---

## 📊 Expo Go vs 실제 빌드 비교

### Expo Go (현재 테스트 환경)

| 기능 | 상태 | 설명 |
|------|------|------|
| 로컬 알림 | ⚠️ **제한적** | Android에서 SDK 53+ 제한 |
| 원격 푸시 | ❌ **불가능** | SDK 53+에서 완전 제거 |
| 에러 메시지 | ⚠️ **표시됨** | 경고만 표시, 앱은 실행됨 |

**현재 상황:**
- 앱은 실행됨
- 로컬 알림 기능이 제한될 수 있음
- 에러 메시지가 표시되지만 앱 자체는 작동

### 개발 빌드 / 프로덕션 빌드 (실제 설치)

| 기능 | 상태 | 설명 |
|------|------|------|
| 로컬 알림 | ✅ **완전 작동** | 모든 기능 정상 |
| 원격 푸시 | ✅ **사용 가능** | (현재 사용 안 함) |
| 에러 메시지 | ✅ **없음** | 정상 작동 |

**실제 빌드에서는:**
- ✅ 모든 알림 기능 정상 작동
- ✅ 에러 메시지 없음
- ✅ 사용자 경험 완벽

---

## ✅ 결론: 실제 빌드에서는 문제 없음

### 이유

1. **로컬 알림만 사용**
   - 이 앱은 원격 푸시 알림을 사용하지 않음
   - 로컬 알림은 개발/프로덕션 빌드에서 완전히 작동

2. **Expo Go의 제한사항**
   - Expo Go는 SDK 53+에서 **원격 푸시 알림**만 제한
   - 로컬 알림은 여전히 작동할 수 있지만, 일부 제한이 있을 수 있음
   - 실제 빌드에서는 이런 제한이 없음

3. **코드 검증**
   - `notificationService.ts` 확인 결과
   - 원격 푸시 관련 코드 없음
   - 로컬 알림만 사용

## 💡 추가 정보

### 로컬 알림 vs 원격 푸시 알림

**로컬 알림 (이 앱이 사용):**
- 앱 내에서 스케줄링
- 서버 불필요
- 오프라인에서도 작동
- 예: "활동 시작 5분 전" 알림

**원격 푸시 알림 (이 앱이 사용 안 함):**
- 서버에서 전송
- FCM/APNs 필요
- 온라인 필요
- 예: 서버에서 보내는 메시지

이 앱은 로컬 알림만 사용하므로, 실제 빌드에서는 완벽하게 작동합니다!
